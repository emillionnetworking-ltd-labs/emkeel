# KEEL-107 — emkeel installs the skills it generates into its own repo

Strategy: none (governance/distribution mechanics — not a product feature)

## Context
emkeel couldn't run its own `/strategy`. The skill is generated by `_strategy_skill()` purely to distribute
to consumers; the emkeel repo never received it because `is_self_repo()` skips **all** distributed wiring
when emkeel applies to itself. The engine works (`emkeel strategy advance` runs in the CLI), but the skill
**prompt** was missing from `.claude/skills/strategy/`, so `/strategy` couldn't launch in the emkeel window
— even though emkeel produces it and governs itself (KEEL-95).

The self-skip is **correct** for files emkeel maintains by hand as SOURCE (AGENTS.md, CLAUDE.md, ci.yml,
jira-transition.yml — clobbering them would be wrong). It's **too broad** for GENERATED skills, which have
no hand-maintained source to overwrite — and which emkeel needs to dogfood its own governance.

## Design (general, not a `/strategy`-only patch)
Distinguish hand-maintained SOURCE from generated skills, and share the rule between the action and the
detection so they can't diverge:
- `SELF_EXEMPT_WIRING` is now **hand-maintained source only** (the skill entry removed).
- `_is_generated_skill(rel)` — any `.claude/skills/**/SKILL.md` emkeel generates (today `/strategy`, any
  future one). No hand source → installed in emkeel's own repo too.
- `self_exempt(rel)` = `rel in SELF_EXEMPT_WIRING and not _is_generated_skill(rel)` — the single predicate
  used by **both** `plan()` (the action) and `wiring_drift()` (the detection). Belt-and-suspenders: even if
  a skill were mistakenly listed in SELF_EXEMPT_WIRING, the `_is_generated_skill` guard still installs it.
- The generated `.claude/skills/strategy/SKILL.md` is committed into the emkeel repo (so `/strategy` runs
  now); future `emkeel update` keeps it in sync.

## Acceptance Criteria
1. `self_exempt` skips hand-maintained source (AGENTS.md/CLAUDE.md/ci.yml/jira-transition.yml) in the self
   repo but NOT generated skills.
2. `_is_generated_skill` matches any `.claude/skills/**/SKILL.md` (general rule, future-proof).
3. In a self repo, `plan`/`apply` install `.claude/skills/strategy/SKILL.md` (verbatim `_strategy_skill()`)
   while still skipping the hand-maintained source.
4. `plan` (action) and `wiring_drift` (detection) share `self_exempt` — no divergence (KEEL-95 lock holds).
5. The skill file is present in the emkeel repo; the engine drives it. Integration test (per KEEL-103).
   Bump 0.1.91; all tests pass.
