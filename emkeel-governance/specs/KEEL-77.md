# KEEL-77 — Documentation model: ADR + docs/ convention in the scaffolded AGENTS.md

## Context
Document handling depended on the prompt (ephemeral), not a durable rule — a doc landed in the
right place only because someone said so. In a real governed repo (em-ecosystem) this let
framework-era docs rot: orphaned snapshots in `docs/` that drifted from the code (ECO-13). There
are seven document types, each with a distinct generation path; the rule must derive from a named
model, not from memory.

## Plan
- `emkeel-governance/adr/0003-documentation-model.md` — record the 7-type taxonomy (dogfooding).
- `src/emkeel/init.py` (`_agents_md`) — add a "Documentation (docs/)" section codifying the docs/
  convention: docs/ = product reference (living); governance never in docs/; code-mirroring specs
  regenerated not snapshotted; docs/archive/ with a status header; dead = delete.
- `tests/test_init.py` — assert the generated AGENTS.md carries the docs/ convention. Bump 0.1.63.
- No hard gate (reference docs aren't gate-able).

## Acceptance Criteria
- `AGENTS.md` generated by `emkeel init` contains the Documentation section (docs/ = product
  reference; governance separate; code-mirroring docs regenerated; archive/ + status header; dead =
  delete).
- ADR 0003 recorded. No new gate added (no change to CI behavior).

## Anti-regression
- Test `test_agents_md_mentions_docs_convention`: the generated AGENTS.md includes the docs/ convention.
